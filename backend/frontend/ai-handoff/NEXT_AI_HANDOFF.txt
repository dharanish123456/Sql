What Has Been Implemented

- Admin pages are wrapped with ProtectedRoute(role="admin").
- ErrorBoundary exists to prevent full app crashes.

2) Data layer and async state
- Service layer is in src/services/ (apiClient + per-domain services).
- React Query is used for server state in migrated pages.
- Mock data remains as placeholder/fallback in src/mock.

3) Observability / UX
- Logger utility used for centralized logging.
- Global error UI components exist (PageLoader, ErrorState).
- Route preloading/preload manager exists to warm lazy routes.

Major Issues We Were Fixing
1) Runtime crash pattern on several pages
- Error seen: Cannot read properties of undefined (reading 'title')
- Root cause: API payload shape mismatch; pages directly accessed data.header.title without guard.
- Fix approach: normalize pageData with fallback to mock data and use safe derived values before render.
- Already fixed pages include:
  - VoiceCallPage
  - OutgoingCallPage
  - IncomingCallPage
  - CallHistoryPage
  - CalendarPage
  - DealsDashboardPage was also reported and had same pattern risk.

2) Sidebar dropdown collapsing after navigation
- Symptom: selecting items (e.g., Dashboard/Email submenu entries) closes parent dropdown unexpectedly.
- Root cause: duplicated links in sidebar HTML; active-link sync could pick a non-submenu duplicate.
- Latest fix in src/utils/adminNavigation.js:
  - Prefer visible matched links.
  - Prefer deepest submenu match for active link.
  - Keep ancestor submenu chain open.
- This specifically addressed Email submenu collapse after click.

Latest Concrete Change (just completed)
- File changed: src/utils/adminNavigation.js
- Added helper: getSubmenuDepth(link, container)
- Updated active link selection logic to:
  - filter visible matches
  - pick deepest submenu link among matches
- Result: when clicking Email page, correct parent dropdown remains open.
